# Домашняя сеть

Документ описывает текущую сетевую инфраструктуру homelab: маршрутизацию, VLAN-сегментацию, физическую коммутацию, беспроводную сеть и DNS. Общие соглашения по хостам и сервисам вынесены в `02-conventions.md`, детали гипервизоров — в `04-proxmox.md`.

## 1. Общая архитектура

Ядром сети выступает **OPNsense**, работающий как единственный маршрутизатор, firewall и DHCP/DNS-сервер. OPNsense запущен виртуальной машиной на мини-ПК под управлением Proxmox VE (PVE-Mini). Весь трафик между сегментами и выход в интернет проходят через него.

Кабель провайдера заходит напрямую в мини-ПК на физический интерфейс WAN. Оба роутера Keenetic работают как беспроводные точки доступа (L2), маршрутизацию не выполняют. Управляемый коммутатор TP-Link раздаёт VLAN тегированным транком на все узлы.

Внешний доступ к публичным сервисам организован через VPS с WireGuard-туннелем до внутреннего reverse-proxy (Traefik): интернет → VPS → WG-туннель → Traefik → внутренний сервис. Удалённый доступ в саму сеть — через AmneziaWG.

## 2. Схема VLAN

Сеть разделена на восемь VLAN. В каждом сегменте OPNsense выступает шлюзом с адресом `.1`, раздаёт адреса через Kea DHCP и резолвит имена через Unbound.

| VLAN | Имя | Подсеть | Назначение |
| :--- | :--- | :--- | :--- |
| 1 | — | `192.168.1.0/24` | Не используется (native/legacy) |
| 10 | **MGMT** | `192.168.10.0/24` | OPNsense, Коммутатор, Omada Controller, PVE-Mini и PVE, PBS, Access Points, MGMT Notebook |
| 20 | **INFRA** | `192.168.20.0/24` | AmneziaWG, Xray |
| 30 | **TRUSTED** | `192.168.30.0/24` | Ноутбуки, ПК, телефоны, доверенные устройства |
| 40 | **DMZ** | `192.168.40.0/24` | Traefik — точка входа публичных сервисов |
| 50 | **SERVICES** | `192.168.50.0/24` | Vaultwarden, Authelia, DockerHost VM, прочие сервисы |
| 60 | **IOT** | `192.168.60.0/24` | Телевизоры, бытовая техника, умный дом |
| 70 | **CCTV** | `192.168.70.0/24` | IP-камеры |
| 80 | **GUEST** | `192.168.80.0/24` | Гостевой Wi-Fi |

Шлюз каждого сегмента — OPNsense (`192.168.<vlan>.1`). DHCP-пулы выделены в диапазоне `.100–.200` каждой подсети.

## 4. Гипервизоры и WAN

Сеть держится на двух хостах Proxmox. Детальная сетевая раскладка хостов, мостов и гостей — в `04-proxmox.md`; здесь только точки, где они стыкуются с сетью.

Кабель провайдера заходит в `nic0` мини-ПК, который чистым мостом (без IP на хосте) проброшен в WAN-интерфейс OPNsense. Белый IP терминируется на WAN OPNsense. Второй интерфейс мини-ПК (`nic1`) — VLAN-транк, несёт все VLAN тегированными на коммутатор.

## 5. OPNsense

Виртуальная машина на PVE-Mini. Единственный маршрутизатор и firewall сети.

**Интерфейсы.** WAN (`vtnet0`) подключён к мосту WAN, получает белый IP от провайдера по DHCP; на WAN включены Block bogon networks и Block private networks. LAN-транк (`vtnet1`) несёт восемь VLAN-интерфейсов (теги 10–80), на каждом OPNsense — шлюз `.1`.

**Kea DHCP** работает на всех VLAN, пулы `.100–.200`.

**Unbound DNS** слушает на всех VLAN-интерфейсах и реализует split-horizon: все `*.kvasok.xyz` резолвятся в `192.168.40.11` (Traefik) через Host Override.

**Firewall — текущее состояние.** Floating-правила форсируют DNS: разрешают DNS-запросы только к Unbound (порт 53 → This Firewall) и блокируют любой внешний DNS (порт 53 → any). Базовый per-interface паттерн на VLAN — блокировка приватных сетей → allow any; поверх него на отдельных сегментах стоят специфичные правила (например, заворот телевизоров в IOT или политика по пирам в INFRA). Anti-lockout — автоправило OPNsense — есть только на интерфейсе MGMT, где висит WebGUI; на остальных сегментах его нет. На текущий момент межсегментный доступ открыт; гранулярная изоляция между VLAN настраивается отдельно.

## 7. Беспроводная сеть

Два роутера Keenetic работают как **независимые точки доступа** (режим «Ретранслятор» — L2-коммутация без маршрутизации). Каждый вещает одинаковые SSID, привязанные к VLAN-сегментам; устройства переключаются между точками по силе сигнала.

**Keenetic Giga (KN-1012)** — точка доступа, `192.168.10.25` (MGMT). Подключена транком в порт 1 коммутатора (native MGMT/10, tagged 30/60/70/80).

**Keenetic Speedster (KN-3012)** — точка доступа, `192.168.10.26` (MGMT). Подключена транком в порт 2 коммутатора (native MGMT/10, tagged 30/60/70/80). В него проводом подключён ПК.

**Привязка SSID к VLAN:** домашний SSID → TRUSTED (VLAN 30), SSID для телевизоров → IOT (VLAN 60).

DHCP на самих точках не поднимается — адреса всем клиентам раздаёт OPNsense. Точки только мостят «тег VLAN ↔ SSID». Management-плоскость точек — в native VLAN (MGMT).

## 8. DNS

Резолвинг для всей сети выполняет **Unbound на OPNsense**. Клиенты в каждом VLAN получают адрес шлюза своего сегмента (`192.168.<vlan>.1`) как DNS-сервер.

Split-horizon: запись Host Override `*.kvasok.xyz → 192.168.40.11` направляет все локальные обращения к публичным доменам на внутренний Traefik. Устройства внутри сети обращаются к сервисам напрямую по локальному адресу reverse-proxy, не выходя в интернет.

Форсирование DNS на firewall гарантирует, что запросы идут только через Unbound: внешний DNS (порт 53) заблокирован, обход резолвера невозможен.

## 9. Внешний доступ

Публичные сервисы доступны из интернета через **VPS**, выступающий точкой входа. Между VPS и внутренним Traefik поднят исходящий **WireGuard-туннель**. Порты на домашнем роутере наружу не пробрасываются — соединение инициируется изнутри.

Путь внешнего запроса: интернет → VPS (публичный вход) → WireGuard-туннель → Traefik (`192.168.40.11`, DMZ) → внутренний сервис (SERVICES / прочие VLAN). Traefik терминирует TLS и маршрутизирует запросы по доменам на соответствующие бэкенды.

## 10. Удалённый доступ (VPN)

**AmneziaWG** (INFRA, `192.168.20.11`) обеспечивает обфусцированный VPN-доступ в сеть извне. VPN-подсеть — `10.8.0.0/24`, трафик клиентов маршрутизируется в нужные сегменты через OPNsense. Детали — в `08-amneziawg.md`.

## 11. Поток трафика (сводно)

- **Выход в интернет:** клиент → шлюз своего VLAN (OPNsense) → WAN → провайдер.
- **Между сегментами:** клиент → OPNsense (маршрутизация + firewall) → целевой VLAN.
- **К локальным сервисам по домену:** клиент → Unbound (резолв в Traefik) → Traefik (DMZ) → бэкенд.
- **Извне к публичным сервисам:** интернет → VPS → WG → Traefik → бэкенд.
- **Извне в сеть (VPN):** клиент → AmneziaWG (INFRA) → OPNsense → целевой сегмент.
